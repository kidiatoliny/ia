# Issue Template

Use this template for each created issue.

```markdown
## Summary
<1 paragraph with the concrete problem and user/system impact>

## Why This Matters
- Business impact: <revenue, compliance, support load, trust>
- Technical impact: <reliability, correctness, performance, maintainability>

## Evidence
- Location: `<path/file.ext:line>`
- Signal: <failing test/log/exception/observed behavior>
- Reproduction:
  1. <step 1>
  2. <step 2>
  3. <step 3>

## Root Cause Hypothesis
<brief, technically plausible explanation>

## Proposed Solution
- <implementation action 1>
- <implementation action 2>
- <implementation action 3>

## Acceptance Criteria
- [ ] <behavioral criterion 1>
- [ ] <behavioral criterion 2>
- [ ] <tests or checks proving fix>

## Scope And Risks
- In scope: <files/modules>
- Out of scope: <explicit exclusions>
- Risks/rollout notes: <migration/backward compatibility/monitoring>
```
